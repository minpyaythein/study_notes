#### Bits and Bytes

1 byte = 8 bits

PC works with the theory of 0 or 1.

Data are represented by sets of bits.

Alphabets are normally OK with 1 byte but Japanese alphabets need 2 bytes to represent all.

10進数 = 箱の大きさが9 = 使える数字が10 (0を含む)

n進数 = 箱の大きさがn-1 = 使える数字がn (0を含む)

###### Subtraction

Addition can be used as subtraction.
負の数を使う必要があります。
これを補数表現といいます。

123 + 876 = 999
     9の補数
123 + 877 = 1000
    10の補数

11001 = 00110 = 1の補数
00110 + 1 = 00111 = 2の補数

Find the answer of 11111 - 11011.
11111 - 11011 = 100
11111 + 00101 = 100100 = 00100

#### 情報理論

10111001111
このファイルを速く送るために圧縮したら、1,0,13,02,14

可逆圧縮：完全に元に戻せること
不可逆圧縮：元に戻しても不完全となること

正確に送るための技術：誤り制御
0と1を間違って送っちゃうこと
パリティチェックで防ぐ：データの誤りを検出する仕組み
偶数パリティ：1010101(1)：The last 1 means the total number of digits is odd. 

平均符号長が短くなるように符号を割り当てる技術：ハフマン符号化 (Huffman Coding)
A:3 times, B: 2 times: C: 1 time
ABABCA
The alphabet with the most appearance should be assigned with shortest number and it goes on.
0 represents A, 10 represents B, 11 represents C
Gives: 010010110
